예제 1: 객체를 만드는 세 가지 방법 객체 생성 · ( ) · 이름 · argument/return
파이썬에서 객체를 생성하는 방법은 크게 세 가지입니다.
( )로 생성: 표현식을 묶어 즉시 사용 가능한 객체를 만듭니다.
이름 부여: 변수에 할당해 이후에도 재사용할 수 있습니다.
함수 인자/반환값: 함수에 넘기거나 함수가 반환한 객체에 바로 메서드를 호출할 수 있습니다. 
# ① ( )를 사용한 객체 생성 → 즉시 메서드 호출 가능
a = (1)
print(a, type(1))   # 1  <class 'int'>

(3 + 4).bit_length()   # 7을 2진수로 표현할 때 필요한 비트 수 → 3

# ② 이름을 부여하여 객체 생성
a = 3 + 4   # Global namespace에 a = 7 등록

# ③ 함수의 argument, return으로 사용
def func(a, b):
    return a + b

func(5, 3).bit_length()   # 반환값 8에 바로 .bit_length() → 4
▶ 출력
1 <class 'int'>
4  ← func(5,3)=8, 8의 비트 길이는 4
예제 2: 실행 방식 기준 — 연산자 / 함수 / 메서드 연산자 · 함수 · 메서드
같은 결과라도 연산자, 함수, 메서드 세 가지 방식으로 실행할 수 있습니다.
연산자는 내부적으로 매직 메서드(__add__, __abs__ 등)를 호출합니다. 
a = 100
b = -5

# ① 연산자 사용 → 내부적으로 int.__add__() 호출
print(a + b)            # 95
print(a.__add__(b))     # 95 (동일)

# ② 함수 사용 → 내부적으로 int.__abs__() 호출
print(abs(b))            # 5

# ③ 객체의 메서드 사용
print(a.bit_length())    # 7 (100을 2진수로 표현: 1100100 → 7비트)
▶ 출력
95
95
5
7
예제 3: 메서드 vs 함수 — self의 유무 self · 메서드 · 함수
메서드는 첫 번째 parameter가 self입니다. 호출 시 인스턴스가 자신의 id를 self에 자동 전달합니다.
함수는 self가 없습니다. 객체에 종속되지 않고 독립적으로 존재합니다. 
# 메서드: parameter에 self가 있음
help(int.__add__)
# __add__(self, value, /)  ← self가 있음

# 함수: parameter에 self가 없음
help(map)
# map(func, *iterables) --> map object  ← self 없음
▶ 핵심 차이
구분 self 여부 호출 방법 예시
메서드 있음 ✅ 객체.메서드() a.bit_length()
함수 없음 ❌ 함수(인자) abs(b), map(f, lst)
예제 4: 중첩 명령 (Nested) 중첩 · nested · 안→바깥
중첩 명령: 함수 안에 함수를 넣는 방식. 안쪽부터 바깥쪽 순서로 실행됩니다.
print(int(input(...))) → ① input() 실행 → ② int() 변환 → ③ print() 출력 
# 중첩 명령: 안쪽 → 바깥쪽 순서로 실행
# ① input('숫자를 입력하세요 :')  → 문자열 반환
# ② int(문자열)                   → 정수로 변환
# ③ print(정수)                   → 출력
print(int(input('숫자를 입력하세요 :')))

# 동일한 코드를 단계별로 나눠 쓰면:
user_input = input('숫자를 입력하세요 :')  # ① '42' (문자열)
number     = int(user_input)              # ② 42  (정수)
print(number)                              # ③ 출력

# 또 다른 예시
print(len(str(12345)))   # str(12345)='12345' → len=5
▶ 실행 흐름
print( int( input('숫자를 입력하세요 :') ) )
                ① 먼저  →  ② 다음  →  ③ 마지막
예제 5: 연쇄 명령 (Chained) 연쇄 · chained · 왼→오른쪽
연쇄 명령: 메서드 실행 후 반환된 객체에 바로 다음 메서드를 호출하는 방식. 왼쪽에서 오른쪽으로 실행됩니다.
단, 반환값이 None인 메서드(inplace 동작, 예: list.append())에는 연쇄 명령을 사용할 수 없습니다. 
a = 'Good Morning'

# 연쇄 명령: 왼쪽 → 오른쪽 순서로 실행
# ① a.upper()  → 'GOOD MORNING'
# ② .lower()   → 'good morning'
# ③ .split()   → ['good', 'morning']
b = a.upper().lower().split()
print(b)   # ['good', 'morning']

# ❌ append()는 None 반환 → 연쇄 불가
b.append('tom')   # inplace: b 자체를 변경, 반환값 없음
print(b)           # ['good', 'morning', 'tom']

# ❌ 잘못된 연쇄 명령 예시
# b.append('alice').append('bob')  → AttributeError: 'NoneType' has no attribute 'append'
▶ 출력
['good', 'morning']
['good', 'morning', 'tom']

⚠ append()는 None을 반환하므로 연쇄 명령 불가!
예제 6: 중첩 vs 연쇄 — 한눈에 비교 nested vs chained · 실행 방향
중첩과 연쇄의 가장 큰 차이는 실행 방향과 구조입니다. 
# 같은 결과를 내는 두 가지 방식 비교

text = '  Hello World  '

# 중첩 명령: 안 → 밖 / 함수 안에 함수
result1 = list(reversed(text.strip().split()))
# strip()→'Hello World', split()→['Hello','World']
# reversed(...)→역순 이터레이터, list(...)→리스트

# 연쇄 명령: 왼 → 오른쪽 / 반환값에 바로 호출
result2 = text.strip().split()
# 여기서 reversed는 메서드가 아닌 함수라 연쇄 불가
# → 변수에 받아서 함수에 전달해야 함

print(result1)   # ['World', 'Hello']
print(result2)   # ['Hello', 'World']
▶ 중첩 vs 연쇄 요약
구분실행 방향구조특징
중첩 (Nested) 안 → 바깥 f(g(h(x))) 함수 안에 함수
연쇄 (Chained) 왼 → 오른쪽 x.f().g().h() 반환값에 연속 호출. None 반환 시 불가
예제 7: bit_length() — 반환값에 바로 메서드 호출 반환값 · 메서드 체이닝 · bit_length
함수의 반환값도 객체이므로 바로 메서드를 호출할 수 있습니다.
bit_length()는 정수를 2진수로 표현할 때 필요한 최소 비트 수를 반환합니다. 
# 반환값이 객체 → 바로 메서드 사용 가능
func(5, 3).bit_length()   # func(5,3)=8 → 8의 bit_length = 4

# bit_length 원리
print(bin(8))              # '0b1000' → 4비트
print((8).bit_length())    # 4
print((7).bit_length())    # 3  (111)
print((100).bit_length())  # 7  (1100100)
print((1).bit_length())    # 1  (1)
▶ 출력
0b1000
4
3
7
1
