파이썬의 3가지 Namespace 계층은?
👆 클릭하여 답 확인
ANSWER
Built-in → Global → Local

상위 순서로 검색하며, 없으면 NameError 발생
💻 코드로 이해하기
실제 파이썬 코드를 통해 Name과 Namespace 개념을 확인하세요.
Namespace 검색 순서 (Read)
Local Namespace
함수 내부에서 선언된 이름 ▶ 함수 내부 명령일 때 시작위치
↓ 없으면 상위로
Global Namespace
함수 외부에서 선언된 이름 ▶ 함수 외부 명령일 때 시작위치
↓ 없으면 상위로
Built-in Namespace
파이썬 내장함수, 예외 이름 (Read Only)
↓ 없으면 NameError ❌
예제 1: 변수 타입과 재할당 Namespace · Dynamic Typing
파이썬은 동적 타입 언어입니다. 같은 이름(name)에 서로 다른 타입의 객체를 할당할 수 있습니다. 변수 이름은 Global Namespace에 등록됩니다. 
a = 10
print(type(a))   # <class 'int'>

a = 'python'
print(type(a))   # <class 'str'>
▶ 출력
<class 'int'>
<class 'str'>
예제 2: 객체 ID와 불변성 (immutability) id() · 객체 동일성
a = 10과 a += 1은 서로 다른 객체입니다. 정수는 불변(immutable) 객체이므로, 값이 변경되면 새 객체가 생성되고 이름이 새 객체를 가리킵니다. 
a = 10
print(a, id(a))   # 10 11645640

a += 1
print(a, id(a))   # 11 11645672  ← 다른 id!
▶ 출력
10  11645640
11  11645672  ← id가 달라짐 (새 객체)
예제 3: Built-in 이름 가리기 (shadowing) Shadowing · del
Global namespace에 Built-in과 같은 이름을 등록하면, 검색 시 Global이 먼저 발견되어 내장 함수를 가립니다(shadowing). del로 Global 이름을 제거하면 다시 Built-in을 사용할 수 있습니다. 
sum = 100
print(type(sum))    # <class 'int'>
# sum(range(1,101)) → TypeError: 'int' object is not callable

del sum              # Global에서 sum 제거
print(sum(range(1, 101)))  # 5050 (Built-in sum 사용 가능)
▶ 출력
<class 'int'>
5050
예제 4: Local / Global Namespace 분리 Local · Global · locals()
함수 내부의 이름(a, b, c)은 Local namespace에 등록됩니다. 외부의 Global namespace와 독립적이며, locals()로 확인할 수 있습니다. 
def func(a, b):
    c = a * b
    print(locals())   # Local namespace 확인
    return c

a = 10
b = 20
c = func(a, b)
print(c)             # 200 (Global의 c)
▶ 출력
{'a': 10, 'b': 20, 'c': 200}  ← Local namespace
200  ← Global c
예제 5: NameError vs UnboundLocalError NameError · UnboundLocalError
NameError: 이름 자체가 어느 namespace에도 없는 경우
UnboundLocalError: 이름은 Local namespace에 등록되어 있지만(컴파일 시점에 x=10 때문에), 아직 객체가 연결되지 않은 상태에서 읽으려 할 때 
def func1():
    print(x)       # NameError: x가 어디에도 없음

def func2():
    print(x)       # UnboundLocalError: x는 local에 있지만 아직 미할당
    x = 10         # 이 줄 때문에 x가 local에 등록됨

# co_varnames: 컴파일 시점에 파악된 local 변수 목록
print(func1.__code__.co_varnames)  # ()
print(func2.__code__.co_varnames)  # ('x',)
▶ co_varnames 출력
()
('x',)
예제 6: 이름 규칙과 pandas DataFrame Name Rule · pandas
dict의 key나 DataFrame 컬럼명도 파이썬 이름 규칙을 따르면 점 표기법(dot notation)으로 접근 가능합니다. 숫자로 시작하는 이름('1c')은 규칙에 어긋나서 x['1c']는 되지만 x.1c는 SyntaxError 발생. 
import pandas as pd
x = pd.DataFrame({'a':[1,2,3], 'b':[4,5,6], '1c':[7,8,9]})

print(x.a)       # 정상: 이름 규칙 준수
print(x['1c'])  # 정상: key 접근
# print(x.1c) → SyntaxError: 숫자로 시작하는 이름 불가
▶ 이름 규칙 요약
✅  x.a      → 이름 규칙 준수 (dot notation 사용 가능)
✅  x['1c'] → key 접근 (항상 가능)
❌  x.1c    → SyntaxError (숫자로 시작 불가)
예제 7: 이름은 다르지만 같은 객체 — id()로 확인 id() · Local vs Global · 객체 공유
이름(name)은 namespace별로 독립적이지만, 그 이름이 가리키는 객체(object)는 공유될 수 있습니다.
Global의 a, b를 함수에 전달하면, Local의 a, b는 이름은 다르지만 동일한 객체를 가리킵니다. 반환된 c도 마찬가지로 Local과 Global이 같은 객체를 공유합니다. 
def func(a, b):             # Local: a, b, c
    c = a + b
    print(id(a), id(b), id(c))  # Local id
    print(locals())
    return c

a = 10                        # Global: a, b, c
b = 20
c = func(a, b)
print(id(a), id(b), id(c))  # Global id
▶ 출력 (id 값은 실행 환경마다 다름)
▶ func 내부 (Local)
{'a': 10, 'b': 20, 'c': 30}
11645640  11645960  11646280

▶ func 외부 (Global)
11645640  11645960  11646280

✔ id(Local.a) == id(Global.a)  → 같은 객체 (10)
✔ id(Local.b) == id(Global.b)  → 같은 객체 (20)
✔ id(Local.c) == id(Global.c)  → 같은 객체 (30, return으로 전달)
🔑 핵심 정리: 이름(Name) vs 객체(Object)
NAMESPACE (이름 공간)
a, b, c 라는 이름은
Local과 Global에 각각 독립적으로 존재
OBJECT (객체)
10, 20, 30 이라는 객체는
namespace에 관계없이 하나만 존재하며 공유됨
Global.a ──┐
             ├──▶ 객체 10 (id: 11645640)
Local.a  ──┘

Global.c ──┐
             ├──▶ 객체 30 (id: 11646280)
Local.c  ──┘  ← return으로 전달됨
