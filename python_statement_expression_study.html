🗺️ Statement vs Expression 한눈에 구분

Statement (문장)
실행되는 명령의 단위
결과값을 반환하지 않음

Simple statement
한 줄(Logical Line)로 구성
assignment, del, import,
return, pass, break ...

Compound statement
다른 statement를 포함, 실행 제어
if, for, while, def, class ...
Expression (표현식)
값(value)을 만들어내는 코드
평가(evaluate)되어 객체 반환

Expression의 종류
Atoms       → 기본 구성요소
Primaries → 속성·인덱스·호출
Operations → 각종 연산
Conditionals → 조건 표현식
Lambdas    → 람다
💡 핵심 구분: Expression은 Statement 안에 포함될 수 있지만, Statement는 Expression이 아닙니다.
a = 10 → assignment statement (값을 반환하지 않음)
10, a + b, abs(-5) → expression (값을 반환함)
예제 1: Simple Statement 종류 simple statement · assignment · del
Simple statement는 한 개의 논리적 줄(Logical Line)로 구성됩니다.
세미콜론(;)으로 여러 개를 한 줄에 나열할 수 있습니다.
가장 자주 쓰이는 것은 assignment(할당)과 expression statement입니다. 
# assignment statement (할당문) — 가장 기본적인 simple statement
a = 10
b = 3

# 세미콜론으로 여러 simple statement를 한 줄에 나열
x = 1; y = 2; z = 3

# del statement — namespace에서 이름 제거
temp = '임시 변수'
del temp

# pass statement — 아무것도 하지 않음 (자리 채우기)
pass

# expression statement — 표현식 자체가 문장이 되는 경우
print(a + b)   # print() 호출은 expression statement
a + b          # 값을 계산하지만 아무것도 하지 않음 (REPL에서는 출력됨)

예제 2: 산술 연산 (Arithmetic Operations) · 연산자 우선순위 ** · * · / · // · % · 우선순위
산술 연산자 우선순위: ** (거듭제곱) > *, /, //, % > +, -
주의: -1 ** 2는 -(1 ** 2) = -1 — **가 단항 -보다 먼저 적용됩니다.
//는 몫(floor division), %는 나머지(modulo)입니다. 
a = 10
b = 3
c = 2

print(a ** b, a * b, a / b, a // b, a % b)
# 1000  30  3.333...  3  1

print(a ** c, a * c, a / c, a // c, a % c)
# 100   20  5.0       5  0

c = 3 * (2 + 5)   # 괄호로 우선순위 변경 → 3 * 7 = 21
print(c)           # 21

# ⚠ 주의: ** 는 단항 - 보다 우선순위가 높음
a = -1 ** 2   # -(1**2) = -1  (NOT (-1)**2 = 1)
print(a)        # -1

b = (-1) ** 2  # (-1)**2 = 1  (괄호로 명시)
print(b)        # 1

100  20  5.0  5  0

예제 3: Arithmetic Conversions (산술 형 변환) int → float → complex · 자동 형변환
서로 다른 타입 간 산술 연산 시 파이썬이 자동으로 타입을 변환합니다.
변환 방향: int → float → complex (더 넓은 타입으로 자동 승격) 
print(type(1 + 3.14))    # int + float → float
print(type(1 + 3j))      # int + complex → complex
print(type('%d' % 10))   # 문자열 포맷 → str

# 수동 형변환 예시
print(int(3.9))     # 3  (소수점 버림)
print(float(5))    # 5.0
print(str(100))    # '100'
▶ 출력
<class 'float'>
<class 'complex'>
<class 'str'>
3   5.0   100
예제 4: 비교 연산 — == vs is · interning == · is · in · interning · hash
==: 값(value)이 같은지 비교
is: 동일한 객체인지 비교 (id가 같은지)
interning: 파이썬이 자주 쓰는 작은 정수(-5~256)와 짧은 문자열을 재사용하는 최적화.
interning 범위 밖(예: 300)은 같은 값이어도 다른 객체가 생성될 수 있어 is가 False가 됩니다. 
# interning 범위 안 (소정수: -5 ~ 256)
a = 10; b = 10
print(a is b, a == b)   # True True  (같은 객체 재사용)

# interning 범위 밖
a = 300; b = 300
print(a is b, a == b)   # False True  (값은 같지만 다른 객체)

# tuple: 같은 값이어도 다른 객체
a = (1, 2, 3); b = (1, 2, 3)
print(a is b, a == b)   # False True

# 짧은 문자열: interning 적용됨
a = "Pizza"; b = "Pizza"
print(a is b, a == b)   # True True

# is로 타입 확인
print(int is type(10))    # True

# 연쇄 비교 (가능하지만 권장 안 함)
a = 10
print(1 < a <= 10)       # True
▶ 출력
True  True
False  True  ← 300은 interning 범위 밖
False  True
True  True  ← 문자열 interning

예제 5: in 연산자 · hash · 탐색 성능 in · not in · hash · list vs set
in / not in: 컨테이너에 해당 값이 있는지 확인. 결과는 bool.
list의 in은 순차 탐색(O(n)), set/dict의 in은 hash 탐색(O(1))으로 빠릅니다.
hash는 불변 객체만 가질 수 있어, 가변 객체(list)는 hash() 호출 시 TypeError 발생합니다. 
a = list(range(1, 10))
print(1 in a, 10 in a)    # True False  (순차 탐색 — 느림)

a = set(range(1, 10))
print(1 in a, 10 in a)    # True False  (hash 탐색 — 빠름)

# hash(): 불변 객체만 가능
print(hash("12345"))       # 숫자 반환 (불변 str)
print(hash([1,2,3]))      # TypeError: unhashable type: 'list'
▶ 출력
True  False  ← list: 순차 탐색
True  False  ← set: hash 탐색 (더 빠름)

예제 6: 논리 연산자 — and · or · not · 단락 평가 and · or · not · truthy · falsy
and / or: True/False가 아니라 앞 또는 뒤의 객체 자체를 반환합니다.
and: 앞이 거짓이면 앞을, 앞이 참이면 뒤를 반환
or: 앞이 참이면 앞을, 앞이 거짓이면 뒤를 반환
거짓(Falsy): False, 0, 0.0, None, '', [], {}, set() / 참(Truthy): 그 외 모든 것 
# and / or 는 bool이 아닌 객체를 반환
a = 10; b = 5
print(a and b)   # 5   (a가 참 → b 반환)
print(a or  b)   # 10  (a가 참 → a 반환)

a = 0; b = 5
print(a and b)   # 0   (a가 거짓 → a 반환)
print(a or  b)   # 5   (a가 거짓 → b 반환)

# not 은 True/False 반환
print(not 0)    # True
print(not 10)   # False
print(not [])   # True  (빈 리스트는 거짓)
▶ 출력
5   10
0   5
True   False   True
예제 7: & · | vs and · or — 연산식 만들기 & · | · and · or · 조건 표현식
and / or: 파이썬 논리 연산자, bool 평가에 사용
& / |: 비트 연산자, pandas 인덱싱에서는 and/or 대신 &/|를 사용
일상적인 조건 비교에서는 둘 다 동일한 결과를 냅니다. 
a = 10

# a가 짝수인가?
print(a % 2 == 0)              # True
print(not (a % 2))             # True  (0은 falsy)

# a가 5보다 크고(and) 짝수인가?
print((a > 5) and (a % 2 == 0))   # True
print((a > 5) &   (a % 2 == 0))   # True  (pandas에서 사용)

# a가 100보다 크거나(or) 0보다 작은가?
print((a > 100) | (a < 0))      # False

# a는 정수이고 b는 문자열인가?
b = 5
print((type(a) is int) & (type(b) is str))  # False (b는 int)

# a가 3의 배수인가?
print(a % 3 == 0)                # False (10 % 3 = 1)
