※ 핵심 원칙: import 뒤에 오는 이름만 namespace에 등록됩니다.
예제 1: import math — 모듈 전체 가져오기 import · namespace · globals
import math는 math라는 이름을 namespace에 등록합니다.
이후 모듈명.함수명 형태로 기능을 사용합니다.
globals()로 현재 namespace에 등록된 이름 목록을 확인할 수 있습니다. 
import math
# 'math'가 namespace에 등록됨
# globals()에서 'math': <module 'math'> 확인 가능

print(math.sqrt(4))    # 2.0
print(math.pi)          # 3.141592653589793
print(math.floor(3.7))  # 3
output
2.0
3.141592653589793
3
예제 2: from math import sqrt — 이름 직접 가져오기 from · import · NameError
from math import sqrt는 sqrt만 namespace에 등록됩니다. math는 등록되지 않습니다.
이후 sqrt(4)처럼 모듈명 없이 바로 사용합니다.
import 뒤에 오는 이름만 사용 가능하다는 원칙을 꼭 기억하세요. 
from math import sqrt
# 'sqrt'만 namespace에 등록됨. 'math'는 등록 안 됨!

print(sqrt(9))     # OK → 3.0
# print(math.sqrt(9))  → NameError: name 'math' is not defined

# 여러 이름을 한 번에 가져올 수도 있음
from math import sqrt, pi, floor
print(sqrt(16), pi, floor(2.9))
output
3.0
4.0 3.141592653589793 2
예제 3: import ... as — 별칭(alias) 사용 as · alias · pandas
import pandas as pd는 pd라는 별칭으로 namespace에 등록됩니다.
이후 pandas라는 원래 이름은 사용할 수 없고, 오직 pd만 사용 가능합니다.
긴 모듈 이름을 짧게 줄이거나 이름 충돌을 피할 때 유용합니다. 
import pandas as pd
# 'pd'가 namespace에 등록됨. 'pandas'는 사용 불가!

s = pd.Series([1, 2, 3])
print(s)
# pandas.Series([1,2,3])  → NameError: 'pandas' is not defined

# from ... import와 as 조합도 가능
from pandas import Series
s2 = Series([10, 20, 30])
print(s2)
output
0   1
1   2
2   3
dtype: int64

0   10
1   20
2   30
dtype: int64
예제 4: NameError — namespace에 없는 이름 사용 NameError · namespace · 주의
namespace에 등록되지 않은 이름을 사용하면 NameError가 발생합니다.
가장 흔한 실수는 from X import Y 후에 X.Y()를 호출하는 것입니다.
등록된 이름은 globals()로 확인할 수 있습니다. 
from math import sqrt

sqrt(4)         # OK: sqrt는 등록됨
# math.sqrt(4)  → NameError: name 'math' is not defined

import pandas as pd

pd.Series([])   # OK: pd는 등록됨
# pandas.Series([])  → NameError: name 'pandas' is not defined

# namespace 확인
print('sqrt' in globals())   # True
print('math' in globals())   # False

NameError: name 'pandas' is not defined
