예제 1: class, instance, self class · instance · self
self는 메서드의 첫 번째 매개변수로, 인스턴스가 메서드를 호출할 때 자신의 id(주소)를 자동으로 전달합니다.
따라서 s1.introduce() 내부의 id(self)는 외부의 id(s1)과 동일합니다. 
class Student:
    def __init__(self, name):
        self.name = name       # 속성 등록

    def introduce(self):
        print(id(self))        # self의 id = 인스턴스의 id
        print(f'안녕하세요, 저는 {self.name} 입니다.')

s1 = Student('철수')
s2 = Student('영희')
print(id(s1), id(s2))   # 서로 다른 id

s1.introduce()           # self == s1
s2.introduce()           # self == s2
▶ 출력
132721663415056  132722229195024  ← s1, s2는 다른 객체
132721663415056  ← id(self) == id(s1) ✔
안녕하세요, 저는 철수 입니다.
132722229195024  ← id(self) == id(s2) ✔
안녕하세요, 저는 영희 입니다.
예제 2: type()과 메타클래스 활용 type() · 메타클래스
type(객체)는 그 객체의 클래스를 반환합니다. 반환된 클래스는 호출 가능하므로, type(10)('123')은 int('123')과 동일합니다. 
print(type(10))          # <class 'int'>
print(type('hello'))     # <class 'str'>
print(type([1,2,3]))    # <class 'list'>

# type(10) → int 클래스 → int('123') 호출
print(type(10)('123'))  # 123  (문자열 '123'을 정수로 변환)

# 클래스도 객체다: type이 클래스의 타입
class MyClass: pass
print(type(MyClass))   # <class 'type'>
▶ 출력
<class 'int'>
<class 'str'>
<class 'list'>
123
<class 'type'>
예제 3: 캡슐화 (Encapsulation) — getter / setter 캡슐화 · _score · getter/setter
속성 앞에 _를 붙이면 보호된 속성(protected)을 나타냅니다 (강제성 없음, 관례).
getter/setter 메서드로 값의 유효성 검사 등 접근을 통제할 수 있습니다. 
class Student:
    def __init__(self, name, score):
        self.name   = name    # 일반 속성
        self._score = score   # 보호된 속성 (캡슐화)

    def get_score(self):          # getter
        return self._score

    def set_score(self, value):   # setter (유효성 검사)
        if value < 0 or value > 100:
            print('잘못된 점수입니다')
        else:
            self._score = value

s = Student('철수', 90)
print(s.get_score())    # 90
s.set_score(150)        # 잘못된 점수입니다
s.set_score(95)         # 정상 설정
▶ 출력
90
잘못된 점수입니다
예제 4: 상속 · super() · 메서드 오버라이딩 상속 · super() · Override
상속(Inheritance): 부모 클래스의 속성과 메서드를 자식 클래스가 물려받습니다.
super().__init__()으로 부모의 초기화를 호출하고, 자식이 같은 이름의 메서드를 재정의하면 오버라이딩(Overriding)이 됩니다. 
class Student:
    def __init__(self, name, score):
        self.name = name
        self._score = score

    def introduce(self):
        print(f'이름: {self.name}, 점수: {self._score}')

class GraduateStudent(Student):   # Student 상속
    def __init__(self, name, score, thesis):
        super().__init__(name, score)  # 부모 __init__ 호출
        self.thesis = thesis

    def introduce(self):             # 메서드 오버라이딩
        print(f'대학원생: {self.name}, 점수: {self._score}, 논문: {self.thesis}')

s1 = Student('철수', 90)
s2 = GraduateStudent('영희', 95, 'AI 연구')
s1.introduce()
s2.introduce()
▶ 출력
이름: 철수, 점수: 90
대학원생: 영희, 점수: 95, 논문: AI 연구
예제 5: 다중 상속 · MRO (Method Resolution Order) 다중상속 · MRO · __mro__
여러 부모 클래스에 같은 메서드가 있을 때 파이썬은 MRO(Method Resolution Order)로 어느 것을 쓸지 결정합니다.
C(A, B)에서 A가 B보다 먼저 나열되었으므로 A의 메서드가 우선합니다. 
class Flyable:
    def fly(self): print('날다')

class Swimmable:
    def swim(self): print('수영하다')

class Duck(Flyable, Swimmable):  # 다중 상속
    pass

d = Duck()
d.fly()
d.swim()

# MRO 충돌 예시
class A:
    def hello(self): print('A')
class B:
    def hello(self): print('B')
class C(A, B): pass

c = C()
c.hello()           # A → MRO 순서상 A가 먼저
print(C.__mro__)    # (C, A, B, object)
▶ 출력
날다
수영하다
A
(<class 'C'>, <class 'A'>, <class 'B'>, <class 'object'>)
예제 6: 다형성 (Polymorphism) 다형성 · 덕타이핑
다형성: 같은 이름의 메서드 호출인데, 객체의 타입에 따라 다르게 동작하는 것.
파이썬은 타입 검사 없이 speak() 메서드만 있으면 호출 가능한 덕 타이핑(Duck Typing) 방식을 사용합니다. 
class Dog:
    def speak(self): print('멍멍')

class Cat:
    def speak(self): print('야옹')

animals = [Dog(), Cat()]

for a in animals:
    a.speak()   # 타입에 따라 다르게 동작
▶ 출력
멍멍
야옹
예제 7: 매직 메서드 · dir() · isinstance() __add__ · dir() · isinstance()
y + 10은 내부적으로 y.__add__(10) 또는 int.__add__(y, 10)을 호출합니다.
dir()은 객체가 가진 모든 속성/메서드 목록을, isinstance()는 타입 확인을 수행합니다. 
x = int(3.5)   # 3
y = x + 10     # 13

# 세 표현은 모두 동일
print(y.__add__(10))        # 23
print(y + 10)                # 23
print(int.__add__(y, 10))   # 23

# dir(): 사용 가능한 속성/메서드 목록
print('__add__' in dir(y))   # True

# isinstance(): 타입 확인
from collections.abc import Iterable
print(isinstance([1,2,3], Iterable))  # True
print(isinstance(10, int))              # True
▶ 출력
23  23  23
True
True
True
예제 8: int 상속과 __new__ __new__ · 상속 · 불변 객체
int처럼 불변(immutable) 객체를 상속할 때는 __init__ 대신 __new__를 사용합니다.
__new__는 인스턴스를 생성하는 단계, __init__은 이미 만들어진 인스턴스를 초기화하는 단계입니다. 
class A(int):              # int 상속
    def __new__(cls, value):
        return super().__new__(cls, value)

x = A(3.5)    # 3.5 → 3 (int 변환)
y = x + 10    # 13
print(y)
print(type(x))  # <class '__main__.A'>

# __new__ vs __init__
# __new__  : 인스턴스 생성 (return 필요)
# __init__ : 생성된 인스턴스 초기화 (return 불필요)
▶ 출력
13
<class '__main__.A'>
